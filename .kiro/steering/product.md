# 제품 개요

MarkSpace는 소규모 폐쇄형(closed) 협업 문서 서비스로, Notion과 유사하지만 필수 기능만 제공한다.
회원 가입(self sign-up) 절차가 없고, 모든 사용자 계정은 단일 admin이 수동으로 등록·관리한다.

## 핵심 기능

- **워크스페이스 기반 협업**: 사용자는 하나 이상의 워크스페이스에 소속되어 owner/member 권한으로
  계층적 markdown 문서를 편집한다. 편집·관리 권한은 워크스페이스 단위로만 존재하며(문서별 개별 권한은
  없음), 문서·첨부·버전·워크스페이스 상세 읽기는 인증된 활성 사용자에게 전역 개방된다(멤버십 무관).
- **버전 관리**: 문서 저장 시마다 스냅샷(버전)이 무한 보관된다. rollback(과거 버전 복원) 기능은 없다.
- **3단계 문서 생명주기**: active → trashed → deleted. 삭제는 그 시점의 서브트리를 "묶음(bundle)"으로
  포착하는 비흡수(no-absorption) 모델을 따른다 — 서로 다른 시점의 삭제는 병합되지 않는다.
- **편집 잠금(lock)**: 동시 편집 충돌을 lock 방식으로 방지한다(CRDT 등 실시간 동시편집은 범위 밖).
- **읽기 전용 공유 링크**: 문서 단위로 외부에 공개하며, 워크스페이스의 `is_shareable` 플래그가 게이트 역할을 한다.
- **첨부/이미지**: 파일로 저장하고(base64 인라인 아님) 워크스페이스별로 격리 보관한다.

## 대상 사용 사례

- 소규모 팀/조직의 폐쇄형 내부 문서 협업 — 초대 없는 self-signup을 금지하고 admin이 계정을 발급하는
  통제된 환경이 필요한 경우.
- 문서 삭제·복구가 빈번한 협업 환경에서, 휴지통 동작(독립 묶음·독립 타이머)이 예측 가능해야 하는 경우.

## 가치 제안

Notion 대비 최소 기능 세트로 구현 복잡도를 낮추면서도, 권한·휴지통·공유 모델의 까다로운 엣지케이스
(묶음 비흡수, 독립 보관 타이머, 재발급 원칙 등)를 명시적 불변식(Invariants)으로 정의하여
구현 간 일관성을 보장한다.
