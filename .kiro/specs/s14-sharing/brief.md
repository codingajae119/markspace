# Brief: s14-sharing

## Problem
문서를 외부에 읽기 전용으로 공개하는 링크가 필요하다. 워크스페이스 `is_shareable`가 게이트이고,
공유는 문서 단위이며, 무효화 이후에는 항상 재발급하는 "재발급 통일" 원칙을 따른다. 링크로 첨부
파일 접근도 허용되지만 게이트/휴지통 상태에 따라 함께 차단된다.

## Current State
s01의 share_link 스키마·s05 is_shareable 게이트·s07 문서 status·s12 첨부 저장 확보. 링크 발급/무효화/
서빙 동작 미구현. 최상위 계층(L6)으로 전체 결합의 종단.

## Desired Outcome
- WS `is_shareable=false`면 링크 생성/활성 불가(REQ-7.1). owner/admin이 게이트 설정(7.2).
- editor+가 공유 가능 WS 문서의 링크 발급/활성(7.3), 문서 단위 읽기 전용(7.4).
- 활성 링크 접근 시 문서 + active 하위 표시(7.5), 새 하위 추가 시 동적 포함(7.6).
- 토글 off/on: 동일 링크 상태 전환(7.7, 토글만 상태 기반 예외).
- 무효화 후 재발급 원칙: 문서 trashed 즉시 무효(7.8), 복구 시 재발급 필요(7.9), WS 게이트 off 시 즉시
  무효·재 on 시 재발급(7.10), INV-8.
- 링크 경유 첨부 다운로드·이미지 로딩 허용(8.4), 게이트 off/문서 trashed 시 파일 접근 함께 차단(8.5).

## Approach
s07 문서 status·s05 게이트·s12 첨부 서빙 위에 공유 링크 서비스를 얹는다(최상위 계층). 무효화는
status/게이트 관찰로 판정하고, 토글만 상태 기반, 그 외는 재발급. 공개 접근 경로는 인증 우회하되
게이트·status·격리로 접근 범위 제한.

## Scope
- **In**: 링크 발급/토글/무효화/재발급, 공개 읽기전용 렌더(문서+active 하위, 동적), 게이트 연동,
  링크 경유 첨부 서빙·차단(8.4·8.5).
- **Out**: is_shareable 설정 UI 소유(s05가 플래그 소유 — 여기선 소비), 문서 상태 전이(s07·s10),
  첨부 저장/격리(s12 — 여기선 서빙만).

## Boundary Candidates
- 공유 링크 생명주기(발급/토글/무효화/재발급)
- 공개 읽기전용 렌더(동적 하위 포함)
- 링크 경유 첨부 접근 게이팅

## Out of Boundary
- WS 게이트 플래그 자체 관리(s05)
- 첨부 파일 저장/아카이브(s12)

## Upstream / Downstream
- **Upstream**: s01(share_link), s05(게이트), s07(문서 status·하위트리), s12(첨부 서빙), s13(게이트 통과)
- **Downstream**: 없음(최상위). s15 전체 e2e 체크포인트

## Existing Spec Touchpoints
- **Extends**: 없음(신규 최상위)
- **Adjacent**: s05(게이트 소유), s12(첨부 소유), s10(status 변화 관찰)

## Constraints
INV-8(무효화 링크 재발급 없이 접근 불가), 재발급 통일 원칙(§4.5), 읽기 전용, WS 경계(INV-6). 한국어.
