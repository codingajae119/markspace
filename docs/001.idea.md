notion과 유사하지만 매우 단순한 필수 기능만 포함한 서비스
- 사용자는 id, password로 로그인
- workspace
  - 사용자는 owner, editor 혹은 viewer 일수 있음 (권한)
  - owner가 사용자들의 권한과 공유 여부 결정
    - 여러 owner 가능
    - owner는 읽기 쓰기도 가능
- admin 계정
  - 사용자 등록, 삭제, 비활동(로그인 금지) 설정
- 파일은 markdown 형식으로 저장
  - 계층적 문서 구조
  - preview창
  - 붙여넣기로 이미지 uploading
  - 파일 첨부
  - 외부에 공유 가능 (공유 가능한 workspace에 속해 있는 경우에)

공유

공유 가능한 워크스페이스에 한해 링크 기반 공개 공유, 읽기 전용

editor 이상이 CRUD
admin이 특정 워크스페이스 멤버가 아니어도 문서 내용에 접근 가능 (admin 권한에 제약 없음)
비활동 사용자 처리: 로그인 금지 시, 그 사용자가 유일한 owner인 워크스페이스라도 editor 및 viewer 활동에 문제 없음. owner 변경이 필요하면 admin이 변경하면 됨.
사용자 삭제 시 데이터: admin이 사용자 삭제 시 삭제되었다고 flag만 변경.
공유 링크 관리: 껐다 켜는 토글 적용
워크스페이스 공유 가능 여부는 owner가 결정. admin도 가능
계층 내 문서 이동/재정렬은 가능해야 함
휴지통/복구 가능해야 함 (30일 후 영구 삭제, 기간은 설정 가능)
버전 기록 / 동시 편집 : UI적으로 편집 시작 button 과 저장 버튼을 두어서 저장시 새로운 버전을 생성하고, 편집 시작시 lock을 두어 동시 편집을 막으면 될 듯함 (UI 적으로 다른 사용자가 편집 중인지 표시)

lock 타임아웃은 도입하지 말고 owner 혹은 admin이 강제 해제 할 수 있도록
편집 취소: 시작했다가 저장 안 하고 나갈 때 → lock 해제 + 변경분 폐기
lock 중 삭제: 문서 삭제도 삭제 flag만 설정하도록 함. 편집 상태와 삭제 상태가 충돌 없음
이전 버전 열람/복원(rollback) 기능은 도입 안함.
버전 보관 정책 : 무한 저장
문서 삭제, 복구는 하위 문서를 포함해서 동작
- 삭제할 때는 하위 문서로 함께 삭제
- 하위 문서만 복구 할 수 없음
이동 범위: 같은 워크스페이스 내만
순환 방지: 문서를 자기 하위로 이동 금지.
공유 링크는 문서 단위 (WS 플래그는 게이트)
공유된 문서의 하위 문서도 링크로 따라 보여짐
WS 공유 플래그를 OFF로 바꾸면, 이미 발급된 문서 링크는 즉시 무효화
사용자 삭제 flag는 flag만 바꾸므로 **재활성화(복구)**도 가능
삭제된 사용자가 버전 히스토리·문서 작성자로 이름이 남음
admin은 단수임 (db에 매뉴얼로 설정)

문서 삭제 상태는 휴지통에 있는 것과 영구 삭제를 분리해야 함. 내가 말한 delete flag는 영구 삭제때 설정됨. 휴지통에 있는 것도 분리된 flag로 관리하든가, 삭제 상태를 활성, 휴지통, 삭제 이렇게 3단계로 나누어서 관리해야 함. 검토해줘.
공유 중인 문서를 삭제(휴지통행)하면 공개 링크는 즉시 무효
공유된 문서의 하위 문서는 개별 삭제 가능함. 휴지통에 있을 때는 부모와 함께 다니지만 활성 상태에서는 따로 삭제 가능함
공유 후 새 하위 문서를 추가하면 자동으로 링크에 포함
WS 플래그 OFF → 링크 무효: OFF 후 다시 ON하면 재발급

자식이 먼저 휴지통에 간후에 부모가 휴지통에 가는 경우는 먼저 간 자식을 따로 관리하지말고 부모와 묶어서 관리
휴지통에서 사용자가 즉시 "완전 삭제" 가능
휴지통 접근·복구 권한: editor 이상 워크스페이스 휴지통 전체 가능
복구(휴지통→활성)하면 링크는 재발급

부모가 휴지통 갈 때 자식을 흡수: 부모의 삭제 시점 기준으로 30일이 재설정
복구 시 묶음 풀림 여부: 부모를 복구하면 흡수됐던 자식도 함께 활성 복귀
수동 완전삭제 : 하위 묶음 통째로 영구삭제

문서 검색은 추후에 정리
첨부/붙여넣기 이미지는 파일로 저장

문서 영구삭제 시 첨부 파일은 삭제하지말고 삭제된 파일 보관 폴더로 이동만.
과거 버전이 참조하던 이미지가 현재 버전에서 지워지면 그 파일을 삭제된 파일 보관 폴더로 이동
읽기 전용 공개 링크에서 첨부 파일 다운로드·이미지 로딩이 되고 WS 플래그 OFF/휴지통행 시 이 파일 접근도 함께 차단
파일 저장 위치·식별: 워크스페이스별 격리 (삭제된 파일 보관 폴더에도 WS 격리 적용)

보관 폴더의 스토리지가 단조 증가는 수용함
보관 폴더는 admin도 볼 수 없음 (system 관리 통해서 수동으로 관리)
파일 복원 경로: 보관 폴더로 가면 영구 삭제된 상태라서 복원 대상이 아님
